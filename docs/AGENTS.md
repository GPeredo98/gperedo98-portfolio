# AGENTS RULES

## Project Goal

Build and maintain a premium personal portfolio that helps generate trust, interviews and freelance opportunities.


## How Agents Must Work

- Respect existing visual style
- Keep code clean and production ready
- Prefer reusable components
- Improve UX when possible
- Keep performance high
- Keep SEO strong
- Do not overengineer simple features


## Priorities Order

1. Existing code patterns
2. ARCHITECTURE.md
3. DESIGN.md
4. User request


## Coding Rules

- Use TypeScript strictly
- Reusable components first
- Keep pages thin
- Move repeated content to /data
- Use semantic HTML
- Accessible components


## UI Rules

- Portfolio must feel premium
- Every section should look intentional
- Strong spacing and typography
- Mobile responsiveness mandatory


## Content Rules

When writing text:

- concise
- credible
- professional
- impact oriented

Avoid generic fluff.


## If Asked to Add Features

Prefer features that improve:

- conversion
- trust
- clarity
- speed
- presentation

Reject unnecessary bloat.


## Examples of Good Additions

- Better project cards
- Case study pages
- Resume download
- Testimonials
- Contact CTA improvements
- SEO improvements
- Analytics


## Examples of Bad Additions

- Heavy gimmicky animations
- Random widgets
- Bloated libraries
- Cluttered layouts
- Generic filler text


## Response Style

When making changes:

- explain what changed
- mention files touched
- mention tradeoffs if relevant
